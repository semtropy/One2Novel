# One2Novel — AI 长篇小说创作工作台

从一句灵感到一本小说。

**你只需要提供想法，AI 帮你完成剩下的所有工作。**

---

## 快速开始

### 下载安装（推荐）

从 [GitHub Releases](https://github.com/semtropy/One2Novel/releases) 下载最新版 `One2Novel-x.x.x-setup-x64.exe`，双击安装。

打开后第一件事：进入**设置**页，填入你的 AI 服务 API Key，然后就可以开始创作了。

> 目前支持 DeepSeek、OpenAI、Anthropic Claude、Google Gemini、通义千问、月之暗面 Moonshot 等多个 AI 厂商。

## 它能做什么

### 核心创作管线

整个创作流程是一条完整的流水线，你只需要在每个节点做最小决策：

```
灵 感 → 故事种子 → 角色生成 → 卷纲蓝图 → 批量写章 → 质检修复 → 导出
```

#### 1. 灵感到故事种子

输入一句话，比如「一个退隐杀手为了救女儿不得不重返江湖」，AI 会生成完整的：

- 故事设定：题材、目标读者、核心卖点、竞品对标
- 多套故事方向供你选择（不满意可以换一批，也可以让 AI 只修某一套的方向或标题组）
- 配套的卷章结构预览

---

#### 2. 角色体系

AI 从大纲中自动提取角色阵容并逐一深化：

- 性格核心、致命缺陷（wound）、发展弧线（arcStart → arcMidpoint → arcClimax → arcEnd）
- 外貌特征、说话风格、内心需求、外在目标
- 与主角的关系定位（relationToProtagonist）、故事功能（storyFunction）
- **角色资源台账**：技能、物品、人际关系等可追踪资产
- **信息差档案**：记录哪些角色知道什么、不知道什么

支持**规划区草稿模式**——AI 生成的角色进草稿区，不直接覆盖已有角色，你确认后才写入正表。成品角色改了，对应草稿自动同步。撤销就是成品变回草稿。

---

#### 3. 卷纲蓝图

AI 自动规划整本书的结构骨架：

- 分卷策略（2-4 卷）
- 每卷的主题、核心冲突、转折点、卷间悬念衔接
- **章节任务书**：每章的具体目标、必须完成的事项、结束时的状态变化
- **节拍表**：卷级节奏曲线可视化
- 卷间**再平衡**：根据已写章节自动调整后续计划

同样支持草稿 → 确认生效的规划区-写作区联动。

#### 4. 章节写作

逐章或批量生成正文：

- **深度上下文组装**：每章写作前 AI 拿到角色硬事实、前文悬念/伏笔状态、卷级任务、风格约束、世界规则、时间线提醒等完整上下文
- **写法引擎注入**：绑定风格档案后按指定写法生成
- **伏笔追踪**：写完自动扫描新设置的伏笔和回收的伏笔
- **时间线提取**：写完自动提取时间事件（事件名/人物/地点/时间点），检测冲突
- **世界规则校验**：自动检查是否违反已激活的世界规则
- **角色硬约束执行**：角色禁止行为列表，AI 审阅时强制检查

---

#### 5. 导演系统（批量自动写作）

不想一章一章点？打开导演模式：

- 从故事设定自动跑完整条链路：方向 → 角色 → 卷纲 → 拆章 → 写作 → 审校
- 断点恢复（中途关了下次接着来，不走回头路）
- 换模型重试（某步不满意换另一个 AI 模型再来）
- 三种推进方式：按重要阶段停下审核 / 自动推进到可开写 / 继续自动执行前 10 章
- 实时查看每步状态、失败原因、下一步建议

---

#### 6. 质检

每章写完后 AI 从 9 个维度评分（每项 1-10 分，满分 90）：

| 维度 | 检查内容 |
|------|---------|
| 开头吸引力 | 前 200 字是否迅速进入情境，是否避免了天气/日常/回顾上章等致命错误 |
| 情节推进 | 是否有实质性状态变化，是否有「推进→受阻→突破」的层次 |
| 人物塑造 | 是否用行动侧面展示而非直接贴标签，行为是否与已建立性格一致 |
| 对话质量 | 每段对话是否至少推动情节/揭示人物/制造冲突/传达信息之一，有无纯寒暄 |
| 悬念设置 | 章尾钩子是否有效（揭示秘密但留更大谜团 / 不可逆决定 / 新危机 / 关系转折） |
| 节奏控制 | 长短句是否交替，全章是否有 2-3 个张力波峰，信息密度是否有起伏 |
| 展示而非讲述 | 情绪是否通过身体反应/动作间接表达，抽象形容词是否被具体描写替代 |
| 语言质量 | AI 痕迹清除：陈词滥调/四字成语堆砌/模板化表达/总结句替代剧情发展 |
| 题材适配 | 根据题材专项检查——悬疑看线索布局和误导设计，言情看 CP 张力和情感节奏，奇幻看世界观一致性，等等 |

低分维度自动触发**技能诊断**：具体症状描述 → 修复建议 → 示例对比。

---

#### 7. 改写工具

选中正文任意段落，一键操作：

| 操作 | 说明 |
|------|------|
| 润色（polish） | 优化语言流畅度和文学质感，保持原意 |
| 扩写（expand） | 增加细节描写、心理活动或场景铺陈 |
| 压缩（compress） | 精简冗余段落，保持核心信息和节奏 |
| 去除 AI 痕迹（fix_ai_traces） | 替换陈词滥调、成语堆砌、模板化表达 |
| 调整视角（rewrite_perspective） | 切换叙事视角 |
| 调整基调（adjust_tone） | 改变情感色彩（更暗黑/更轻松等） |

另有**章节诊断**（workspace diagnosis）：AI 扫描整章，输出诊断卡片——每条标注问题段落索引、严重度、推荐改写操作。

---

#### 8. 导出

支持 4 种格式：

| 格式 | 说明 |
|------|------|
| **EPUB 3.0** | 完整 EPUB 规范，含目录/导航/样式，直接导入微信读书、Kindle 等 |
| **TXT** | 纯文本，带章节目录 |
| **Markdown** | 带 frontmatter 元数据和锚点目录 |
| **JSON** | 结构化完整数据（含角色/卷纲/元信息），便于二次开发 |

导出前可预览标题、题材、总字数、已完成章数等统计信息。

### 辅助功能

#### 写法引擎

- 从你的范文文本中提取写法特征（叙事节奏、对话风格、语言习惯、反 AI 规则等）
- 保存为**风格档案**，提取结果形成可视化特征池，可逐项启用/停用/编辑
- 支持**书级绑定**（整本小说用一个风格）和**章级绑定**（不同章节不同风格）
- 写作时 AI 按权重编译注入风格约束，驱动生成方向

#### 伏笔系统

- 写完每章自动扫描新设置/回收的伏笔
- 完整生命周期追踪：已设置 → 已部分回收 → 已回收 → 已逾期
- 逾期伏笔自动提醒
- 手动增删、编辑伏笔条目，指定目标章号

#### 时间线

- 写完每章自动提取时间事件（事件名、涉及人物、地点、时间点）
- 冲突检测：同人同时出现在两地、事件顺序颠倒、时间跨度不合理
- **写前提醒**：写下一章时自动加载——逾期未发生的伏笔 + 未来 5 章内即将发生的时间线事件
- 同章重提幂等（不会生成重复事件）
- 前端颜色图例区分事件状态，支持手动重提

#### 世界规则

- 创建结构化世界观规则，按类别分组
- 按章节激活/停用规则（不同卷可适用不同规则集）
- 写作时自动检查正文是否违反活跃规则
- 冲突检测 + 手动解决入口

#### 分镜系统

- 为每章创建场景卡片（SceneCard）
- 规划每个场景的视角、冲突类型、情绪强度
- 可视化编排面板

#### 创作助手（Chat）

- 内置对话式 AI 助手
- 用自然语言描述需求
- 支持追问、细化、执行动作（创建小说/生成大纲等）

#### 设置

- 多厂商 API Key 配置（DeepSeek / OpenAI / Anthropic / Gemini / 通义千问 / Moonshot）
- 一键测试连接
- 创作偏好：写作视角、叙事节奏、风格基调、偏好题材、默认章数字数/卷数
- 默认模型选择（可按厂商+模型组合配置）

## 桌面版说明

### 数据存储

所有数据保存在你的电脑上：

```
C:\Users\<用户名>\AppData\Local\One2Novel\data\
```

- 小说、角色、章节等核心数据存在 `dev.db`（SQLite 数据库）
- API Key 和偏好设置存在 `user-preferences.json`
- 卸载时选择「保留数据」则不会删除这些文件

### 自动更新

桌面版启动时会自动检查更新。有新版时会在启动页提示下载。

## 开发说明

本节面向开发者。

### 技术栈

| 层 | 技术 |
|----|------|
| 前端 | React 19 + Vite + Tailwind CSS + Tiptap 编辑器 + Zustand + TanStack Query |
| 后端 | Express 5 + Prisma 7 + SQLite + LangGraph |
| AI | DeepSeek / OpenAI / Anthropic / Gemini / 通义千问 / Moonshot |
| 桌面 | Electron 35 + electron-builder |
| 包管理 | pnpm monorepo（client / server / shared / desktop） |

### 本地开发

```bash
# 安装依赖
pnpm install

# 启动开发环境（server:7456 + client:7457）
pnpm dev

# 类型检查
pnpm typecheck

# 构建桌面安装包（本地）
pnpm dist:desktop:nsis

# 发布桌面 Beta 版到 GitHub
pnpm publish:desktop:beta
```

端口被占用时：`taskkill /F /IM node.exe`

### 项目结构

```
One2Novel/
├── client/          # React 前端
│   └── src/
│       ├── pages/       # 页面组件
│       ├── components/  # UI 组件
│       ├── api/         # API 调用
│       ├── lib/         # 工具函数
│       └── stores/      # 状态管理
├── server/          # Express 后端
│   └── src/
│       ├── app/         # 路由和 HTTP 层
│       ├── modules/     # 业务模块
│       │   ├── novel/       # 核心创作管线
│       │   ├── style/       # 写法引擎
│       │   ├── payoff/      # 伏笔系统
│       │   ├── timeline/    # 时间线
│       │   └── creativeHub/ # 创作助手
│       └── platform/    # 基础设施
├── shared/          # 共享类型和 Schema
├── desktop/         # Electron 桌面运行时
└── scripts/         # 构建和发布脚本
```

### LLM 配置

在项目根目录创建 `.env` 文件：

```env
DEEPSEEK_API_KEY=sk-your-key-here
DEEPSEEK_BASE_URL=https://api.deepseek.com/v1
DEEPSEEK_MODEL=deepseek-chat
```

也支持 OpenAI、Anthropic、Gemini 等厂商，详见 `server/src/platform/config/env.ts`。
